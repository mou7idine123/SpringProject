Application de Gestion de Scolarite

Documentation technique complète — API REST Spring Boot + JWT

1. Présentation

Application REST pour administrer un établissement scolaire : gestion des étudiants, enseignants, modules, inscriptions et notes avec authentification JWT.

Technologies : Java 17  |  Spring Boot 3.2.5  |  Spring Web  |  Spring Data JPA  |  Spring Security  |  MySQL  |  JWT (jjwt 0.11.5)

3. Configuration MySQL

spring.datasource.url=jdbc:mysql://localhost:3306/scolarite_db
spring.datasource.username=root
spring.datasource.password=root
spring.jpa.hibernate.ddl-auto=update

La base scolarite_db et toutes les tables sont créées automatiquement au démarrage.

4. Authentification JWT — Comment se connecter

L'application utilise JSON Web Token (JWT). Il faut d'abord se connecter pour obtenir un token, puis utiliser ce token pour chaque requête protégée.

1 Se connecter — obtenir le token

MéthodeURLAuth requise
POST/api/auth/loginNon — public

Corps de la requête (JSON) :

{
  "email":    "admin@scolarite.com",
  "password": "admin123"
}

Réponse en cas de succès (200 OK) :

{
  "token": "eyJhbGciOiJIUzI1NiJ9.eyJyb2xlcyI6IltST0xFX0FETUlOXSIsInN1YiI6ImFkbWluQHNjb2xhcml0ZS5jb20iLCJpYXQiOjE3MTcwMDAwMDAsImV4cCI6MTcxNzA4NjQwMH0.XXXXX",
  "type":  "Bearer",
  "email": "admin@scolarite.com",
  "role":  "ROLE_ADMIN"
}
Mauvais email ou mot de passe : 401 Unauthorized — "Email ou mot de passe incorrect."

2 Utiliser le token dans Postman

Dans chaque requête protégée, ajouter le token dans l'en-tête HTTP :

Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

Dans Postman :

OngletChampValeur
AuthorizationTypeBearer Token
AuthorizationTokenColler le token reçu au login
Token valide : Accès autorisé — 200 OK ou 201 Created
Sans token ou token invalide : 403 Forbidden

3 Durée de validité

Le token est valide 24 heures. Après expiration, refaire un login pour obtenir un nouveau token.

13. Lancer l'application

# Etape 1 : créer la base MySQL si elle n'existe pas
mysql -u root -proot -e "CREATE DATABASE IF NOT EXISTS scolarite_db;"

# Etape 2 : libérer le port 8080 si nécessaire
fuser -k 8080/tcp

# Etape 3 : lancer l'application
./mvnw spring-boot:run

L'API est disponible sur : http://localhost:8080
